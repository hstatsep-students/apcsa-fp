Unit 3: Boolean Expressions and If Statements

IF statements + if... ELSE statements

How do if statements work?

We use if statements when we want to execute code ONLY if some condition is met. If statements have two part: the condition and the body (the code that get executed if the condition is met): if (condition){ //code } We can also use if statemnets to check whether one value is equal to another in conditions using (==) when these values are number. Some other ways we use it to compare primitive and reference values is using Relational Operators. These include:
  • Equal: ==
  • Less than: <
  • Less than OR equal to: <=
  • Greater than: >
  • Greater than OR equal to: >=
  • Not equal to: !=

What about if...ELSE statements?

Else statements are executed when the condition in the if statement is false. It ALWAYS follows an if statement. We can also combine else statements and if statements into ‘else if’ statements to create longer blocks which has more than two options.

Booleans and Truth Tables

Can Booleans statements have more than one condition?

Yes by using nested conditionals. We can make much more complicated branching logical processes by multiple if and else statements inside others like a flowchart with multiple splits

What are boolean operators?

It is another way to represent multiple conditions and it allows us to combine boolean variables into more complex statement which can still evaluate to only true or false.These include:
  • NOT ! Takes the opposite
  • AND && Both are true
  • OR || One or both is true

Booleans operators have an order in which they should be evaluated. The order !, &&, and lastly ||.

What are Truth Tables?

It is a way of looking at all possible values of the variables and determining the value of the whole statement. Truth tables are an excellent way to identify boolean identities which means finding pairs of statements which are equivalent. An exmaple, would be 2 conditions, A and B. A could be something like ‘x == 45’ or ‘x < 22’

Evaluating

What is short-circuit evaluation?

Short-circuit Evaluation is a shortcut for evaluating boolean statements. There are two kinds of short-circuit evaluation:
  • OR - if two boolean statements are connected by the || operator, and the first statement is true, we don’t need to evaluate the second statement, because the whole thing will be true, regardless of what the second statement is:
    if (true || doesn’t matter)
    {
    //has to be true
    }
  • AND - if two boolean statements are connected by the && operator, and the first statement is false, we don’t need to evaluate the second statement, because the whole thing will be false, regardless of what the second statement is:
    if (false && doesn’t matter)
    {
    //has to be false
    }
Short-circuiting also prevents dividing by 0 errors in a boolean statement by setting the first condition to not equal zero.

What is De Morgan’s Law?

De Morgan’s Law basically simplifys booleans statements. This law includes:
  • Inverting Boolean Statements: !(A || B) is equivalent to !A && !B
  • Opposite of Less Than and Greater Than:
    • opposite of ‘less than’ is ‘greater than or equal to’
    • opposite of ‘greater than’ is ‘less than or equal to’.

What about comparing objects?

We cannot use (==) in objects because (==) only refers to memory locations. Objects have built-in methods to compare like the .equals() method. Remember this refers to wrapper classes too

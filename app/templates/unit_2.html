Unit 2: Using Objects

Class Data type: Strings

What are class Data Type?

Class Data Type (a.k.a. Reference Data Type) can hold many pieces of data at one time, and even many different types of data if you need them to. Programmers can define their class data types, and we’ll learn that later in the course. When we declare a class data type, Java allocates a space in memory called a reference. It only holds the memory location of the actual data and It does this because the data could be large. Class data types have methods that allow us to change or print out the data stored in them.

What are Strings?

Strings are sequences of characters and are immutable. This means that once we set up a String in memory we can't actually change it - all we can do is make the variable point at a different location in memory.When strings are stored in memory, they’re stored as a series of chars which is a primitive data type, representing one character. The chars in a String have indexes, starting from 0, so we can access them individually:
I n d e x
0 1 2 3 4
Since the index of the first character is 0, the index of the last character is length of the string-1 and trying to access any characters outside of this range will cause an index out of bounds exception.

What methods do strings have?

Below are important String Methods. Let String a = "index";
  • .length()
    • Returns an int, the number of chars in the string
    • EX: a.length() = 5, even though the last index is 4
  • .equals(String otherString)
    • Tests if data stored in otherString is the same as this one
    • Returns true or false
  • .indexOf(String str)
    • Returns the index of the first occurrence of str in the calling String
    • EX: a.indexOf("d") = 2
    • Note that indexOf returns -1 if the string isn't found
  • .substring(int firstOneYouWant, int firstOneYouDontWant)
    • Returns the substring from the first int up to but not including the second
    • EX: a.substring(1,4) = "nde"
    • Be super careful about the indices, trying to access a letter outside of the range 0 to length -1 will result in an index out of bounds error!
    • to create a single character substring at position index, we can use .substring(index, index+1)
  • .substring(int firstOneYouWant)
    • Returns the substring from the specified index to the end
    • EX:a.substring(2) = "dex"
  • .toUpperCase()
    • Transforms the string to uppercase
    • EX: a.toUpperCase() = "INDEX"
    • Not tested on the AP exam
  • .toLowerCase()
    • Transforms the string to lower case
    • EX: a.toLowerCase() = "index"
    • Not tested on the AP exam
  • .compareTo(String otherString)
    • Returns 0 if the strings are the same
    • Returns a negative value if this string comes before otherString alphabetically
    • Returns a positive value if this string comes after otherString alphabetically

String Concatenation

String Concatenation basically glue two Strings together. It takes two different strings and print they out using the (+) operator. An example would be:
System.out.println("Hello" + "World");
> HelloWorld

 When concatenting, it is important to be aware of is that if Java is already dealing with Strings then it will take any primitive value because this might happen:
int x = 1;
int y = 3;
System.out.println("The sum is: " + x + y);
> The sum is: 13

 Once Java works with Strings, it continues to look at things as Strings, and interprets ‘+’ as a concatenation operation. However this issue can be solved by using the order of operations:
System.out.println("The sum is: " + (x + y));
> The sum is: 4

The parentheses tell Java that it should add the x and y first.

Escape Characters

Because Strings uses quotation marks escape sequence to indicate to Java when we want to print out a quotation mark.
  • To include a literal quote in a String: \"
  • To include a new line in a String: \n
  • To include a tab in a String: \t
  • To include a literal single quote in a String: \'
  • To include a literal backslash in a String: \\

Classes, Objects, Constructors, and Methods

What is a class?

Classes are template for an object. Specifically the datatype of the object. Class has the instructions to make the actual object. It determines what states will be possible for an object like:
  • What types of data it will store?
  • How many?
  • What restrictions there will be on the values?

What are Objects?

An object in java is something we can use in code that has both state which is the actual value of data that are stored in the memory at the location of this object and behaviors which is action which objects can perform.All objects are store using reference memory and are Specific Instance of the class. When declaring Objects we need the keyword new to T tells Java that we need to reserve some space in memory for the new object we are creating. THE STRING DATA TYPE IS THE ONLY EXCEPTION.

What are constructors?

A constructor is a special method specified in the class that sets up the object as we want (it is always the same name as the class name) and one class can have multiple constructors. Other than the first constructors the rest allows us to pass in info in the form of parameters. The values we pass in as parameters are referred to as actual parameters as opposed to formal parameters defined in the method signature. This means our constructor gets a copy of that value, not the actual value itself, or a copy of the reference to that value if it is reference data Java uses something called a signature whenever a constructor or method appears in source code to identify what it has to call. The signature is made up of the name and the parameter list. Java can still tell which constructor it needs to use though as they all have different numbers of parameters and have the same name. The order of parameters matters.

What are methods?

Methods determine the behavior of a class. When we call a method, we are running code stored elsewhere - in the class for the object we created. There are two types of methods we can call: void and non-void methods. Void methods perform a function BUT don’t give us any info back while non-void methods RETURN a value which we capture using a variable or use in an expression like a print statement. Like constructors, we need to match the signature of a method we are calling by getting the name of the method, as well as the number, order, and types of parameters right. Methods can also be overloaded and have the SAME name but DIFFERENT parameters.

2 specific kind of class: Wrapper and Math Class

How do wrapper class work?

Wrapper classes are ways of storing primitive data types in a class. There is one built in for each primitive data type--class Integer for ints, class Double for doubles, etc. Because Integers and Doubles are objects, we can use different methods like the .compareTo() method which are used to compare their values or the .equal() method.

What is Autoboxing and Unboxing>?

The con of wrapper class was the need to convert between the primitive data type and the wrapper class but now newer Java can Autobox which converts a primitive to an object or Unbox which converts objects back to primitive.

What is the Math Class?

The math class is extremely useful to us when performing calculations in Java. What's special about these methods is that they are all static methods. We don’t have to declare a new Math object to use them (like Math m = new Math(), with m.random()). You can get to these methods by just using the class name (Math), and the dot operator. Below is an list of math class methods:
  • Math.abs -- an overloaded method:
    • takes one double as a parameter and returns the absolute value as a double.
    • double x = 9.6;
      double dist = Math.abs(x);
      System.out.println(dist);
      > 9.6
  • Math.pow:
    • takes two double as inputs and returns the first raised to the second as a double
    • int a = 2
      int b = 6
      double answer1 = Math.pow(a, b);
      System.out.println(answer1);
      > 64.0
  • Math.sqrt
    • Takes a double as input and returns its square root as double
    • double answer2 = Math.sqrt(answer1);
      System.out.println(answer2);
      > 8.0
  • Math.random
    • does not take any input, and returns a random double between 0 and 1
    • double answer3 = Math.random();
      System.out.println(answer3);
      > 0.23487...
    • Random numbers can be very useful in all sorts of applications, but often we want a random integer in a given range, not a random decimal between 0 and 1. We can transform it like this, where we know the range of numbers and the minimum number
      : double ran = Math.random();
      int range = 100;
      int min = 1;
      double answer = (int) (ran * range) + min; // follow order of operations
      System.out.println(answer);
      > 24

Unit 5: Using Objects

Void Methods

What are void methods?

These methods do not have return statement. Void methods are usually formatted like: public static void doStuff() {//code}. The Public indicates that other classes can access the method. The Void indicates that the method doesn’t return anything. The Void can be replace by int, double, boolean, and string to show what data type is getting returned. Public methods must be declare OUTSIDE of the main method.

Parameters

What are parameters?

Parameters are inputs for methods. There are two kind of parameters: formal and actual.

What is formal parameter?

A formal parameter is a local variable in a method header that holds the data sent in. The data type of the parameter makes a huge difference in what the method can do. Formal parameters are declared in the parentheses after the method’s name and need to have their own names.
Formal Parameter Rules:
  • As many as you need
  • Any combination of data types
  • Values are sent in from main (or the method calling the method)
  • The values being sent in are called arguments (or actual parameters)

What is actual parameters?

An actual parameter (a.k.a argument) is the data sent to a method. The data type of the argument MUST match the data type of the formal parameter.

What's the difference between actual and formal parameters?

Formal parameter is the name of a value inside a method. An actual parameter or argument is the name of the value on the outside of the method. REMEMBER THAT ORDER AND DATA TYPE OF ACTUAL AND FORMAL PARAMETERS MUST MATCH.

What the different between Primitive and Class parameters?

Because primitives are stored in memory directly, if they are changed in a void method, those changes are not preserved while while because class data types are stored as references, if they are changed in a void method those changes are preserved ecept for Strings since strings are immutable.

Return Methods

What are return methods?

Methods with return statements which sends a value back to the calling method, and thus provide output which can be stored as a variable. A METHOD CAN ONLY HAVE ONE RETURN VALUE. Once java encounters the return statement, Java ignores anything further after the return statement. To catch the info for the return value, we can store it in a variable, print it, or use it in a boolean statement. If your return a value inside a loop, it ends the loop. You cannot overload a methods with more than one return statements.

To make classes from scratch

Classes are really useful for representing things with multiple data attached to them. When we create a new class, keep in mind:
  • The class name is capitalized
  • The class should go in its own file in the same directory as the rest of our files for a given project
  • The file name should match the class name exactly, followed by a .java extension
We will create a new class by creating a new file and declaring our class as public. To run any java file, we need the static void main method presented. Since we want to keep each class we make separate in a new file, we usually create a runner file that has this main method in it. Then we can instantiate the new object which is when we create a new space in memory to hold a reference to a class. After we can start saving data by declaring the variable for each attribute. Make these variables private so other codes in other classes can not change their values. If we do want to access private variables, we need to write an accessor method that returns the private variable. If we want other files to change the private variables, we need to write a mutator method which is a void method with a parameter passed which defines the behavior of updating an instance variable. The next part is constructors.

Contsructor

To create an object we need a Constructor. If you do not have one, Java adds a default one for us with no parameters. Our Constructors are declared public. Arguments are passed to constructors using call-by-value which copies all the arguments that are made and these are what are used in the constructor.

.toString() Method

Lastly, we shoudl built a toString() method. The toString() method tells Java how to represent objects of a class as strings. It is always public, and always returns a String.

Building on after the Basics

More info on constructor

  • We can use mutator methods in our constructor
  • we can overload our constructors
  • this keyword to have our default constructor actually call our most detailed constructor and populate it with basic info we desire. This keyword allows one method to call another within its class
  • By changing the number of arguments a method has, or changing its signature, Java knows which constructor to use in which situation, and you can chain them together using overloading
  • Every class in Java has to have a constructor so that instances of that class can be created. If we do not add a constructor to our class, Java adds one for us which is the default constructor. This constructor has no parameters and sets all the member variables to default values. This means all class data types are set to null, ints are set to 0, doubles to 0.0, and booleans to false

Static modifers

What are Static modifers?

When the static modifier is added to a variable or method, it means that only one copy exists for all objects of that class. This means that all objects of this class share access to the static entity. Steps to coding Static modifers:
  1. create the static variable and add another non-static variable which will be an instancevariable
  2. need to add code to our constructor to change the static variable and set the instance variable equal to the static variable
  3. in your .toString method, return the value of the static variable by using the instance variable
Static modifers can be used an a counter or constant like Integer.MAX_VALUE.

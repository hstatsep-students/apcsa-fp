Unit 1: Primitive Types

Input and output in Java

What is Input?

Input is pulling information into the computer. This works IF you have a spot in memory to hold this value a.k.a variable.

How do we input in Java program?

We use scanners. Scanners allow us to take user input and incorporate it in our program. Scanner are formatted like: Scanner scan = new Scanner(System.in); and when a scanner take user input, it usually read as a string and stores as a variable

What are variables?

Variables are like containers in memory. When we declare a variable, we tell the computer to allocate some space in memory for something. We tell it what the name of the thing will be so it knows how to find it later, and what the type of the thing will be, so it knows how much space to set aside Variables can be changed. When we put a value in the variable, we initialize the variable (before you can put the variable in a string, you must initialize it). It can be changed to a different value later using an assignment operator (=). Rules of naming variables:
  • Only use letter/numbers + special characters ( _ ) and ( $ ). NO SPACE OR OTHER CHARACTERS ALLOWED.
  • Variables CANNOT start with and number [Ex: num1 is ok but 1num is not ok]
  • Cannot use name that already have a meaning in java.[ Ex: a variable cannot be named string]

How to output in Java?

To output in Java, use System.out.print() which will print on the moniter OR System.out.println() which will prints to the monitor and then prints a blank line. Order matters and all commands in Java end with a semicolon, and if we forget the semicolon we will also get an error when we try to compile

Data types

Why is it important for the computer to know what kind of input you’re giving it?

It is Important to know what kind of input you are inputting so it knows how to deal with the data.Variables in Java are always assigned a type which tell Java:
  • What possible values can be stored - this is called a domain and is important as Java needs to know how big a box in memory to allocate for the input.
  • What operations can be performed on the data - for example can two variables of this type be added together?

There are two kinds of data types in Java: Primitive and Reference Data Type

What are Primitive Data Types?

These data types are stored directly into the memory and has a fixed size. There are 8 primitive data types in Java: byte, char, short, int, long, float, double and boolean. These data types act as the basic building blocks of data manipulation in Java. The three kinds of Primitive Data Types we used most often are:
  • int - used for storing an integer value. ints are pretty small in memory.
  • double - used for storing a decimal value (a number which is not an integer). doubles take up more space than ints because of the decimal component.
  • boolean - used for storing a true or false value, in memory this value can be represented by a single bit of memory, stored as a 1 or 0.
Because boxes for primitive data type are a fixed size, it means there are limits to what they can store, even with the right type of variables. To access the largest + smallest int value which can be stored you can set int equal to Integer.MAX_VALUE or Integer.MIN_VALUE (these are special final values which can be accessed in any java program). In Java, You should add the word final into a variable declaration/initialization statement when you don’t want to keep typing a really long number. This final modifier locks in the value of the variable permanently so that it cannot be changed by any other parts of your code.

What are Reference Data Type?

Reference data are stored as references to complex data structures. Reference data types can store several different pieces of information (e.g. a String with many different characters). Because of this, the value doesn't get stored directly in the variable like with a primitive, instead, a reference to another place in memory is stored. An example would be String - they take up lots of space in memory because they have to store each character separately. We use Strings for anything we won’t do calculations with, like a street address.

Number calculations

How to do calculations in Java?

To do calculations in Java using arithmetic operators which include: ‘+’ (addition), ‘-’ (subtraction), ‘*’ (multiplication), and ‘/’ (division). Java follows the order of operations. When you divide integers, avoid dividing by 0 because dividing by 0 is mathematically impossible so while the code will compile correctly it will result in an ArithmeticException error when it is executed, and look out for double and integers. For example, if you have two int values like 5 and 2. When you divide 5 by 2 your correct answer would be 2.5 but the program is only going to print out the value 2. To solve this issue you should set the answer to a double and then multiple it by 1.0.
int x = 5;
int y = 2;
double z = 1.0 * x / y;
System.out.println(z);
> 4.5

In Java there are also Compound Assignment Operators which are used in place of the normal ‘=’ assignment operator and change the value of a variable. We can create these operators by combining any one of the arithmetic operators with the ‘=’ sign. In code, these would look like:
int x = 3;
x *= 7; // The value of x is now 21
x -= 3; // The value of x is now 18
x /= 6; // The value of x is now 3

There are also two handy unary operators we can use to increment (increase by 1) or decrement (decrease by 1) the value of a variable:
int x = 9;
x++; // x is now 10
x--; // x is now 8

Modular Divison

In Java, there is a calculation called, Modular Division which is remainder division. The modular division is focused on finding the remainder. When we do modular division we say “Six mod 2 is zero”, because there is no remainder. To use modular division use, the (%) operator. Modular works even if the first number is less than the second number. We will get an error if we try to divide by 0 in modular division.

Rounding Doubles

If we want to round a positive double to the nearest integer we can add 0.5 before casting to an int. If the decimal part of the number is greater than or equal to 0.5 then the integer part will be increased by 1 before the cast. If the decimal part is below 0.5 the integer part won't be affected. If we wanted to round a negative number we would have to subtract 0.5 rather than add it.
Double + or - 0.5 Cast to an Int
2.1 2.6 2
2.7 3.2 3
-2.1 -2.6 -2
-2.5 -3.0 -3

Casting

Casting is the process of converting one data type to another. For example, if we want to change an integer to a double we can just cast the variable to a double: double z = (double) x / y; There are two kinds of conversion: Widening and Narrowing Conversion Widening conversions move a data type into a bigger box. For example, you would use wisening conversion when putting an int into a bigger double box. Narrowing conversions move a data type into a smaller box. If we don’t explicitly cast when moving from a larger data type to a smaller one, we’ll get an error telling us there's a “Possible Lossy Conversion” when we try to compile (The phrase "lossy conversion" means we're trying to convert data in a way that loses some of the information).
